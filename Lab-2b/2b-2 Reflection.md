# 2b-2 Introduction to Bash Scripting & System Automation 

This lab introduced me to Bash scripting, something that is still going to take awhile to get my head wrapped around, I started off with something simple like making a directory in Bash using `mkdir`, creating a text file using `touch` inside the directory and have it output something using `nano` to firstly edit what is inside the text file and then using `cat` to take a look at the file. The difference between copy (`cp`) and move (`mv`) in bash is that `cp` would leave the file in whichever directory is in, while creating a new file of the copied file in the desired location, whereas `mv` would take the file from whichever directory it is in, and would create the same file into the desired location, and removing it from whichever directory it was originally in.

Transitioning into making an executable to be able to output a custom message, I started off with creating using `touch` to make hello_world.sh (which I learned later that I did not have to do), followed by editing it using `nano`, from here I had to learn what `#!/bin/bash is`, [GeeksForGeeks, 2026](https://www.geeksforgeeks.org/linux-unix/shell-scripting-define-bin-bash/) refers to it as a 'shebang', it would start off with the special character `#!`, followed by the path to the interpreter, which in this case is bash. It could theoretically be used without a shebang, since it would default to the default terminal which is bash, but it may run into some issues on some systems, which is why it is always placed at the beginning of these scripts, to specify that bash should be the interpreter of the code. After which I just made a simple `echo` message to personalise my script output. I then ran `chmod 777`, as files that are just created may not have the execute permission onto anyone, so even if I am using my admin profile, I still would not be able to execute it as there are no permissions on the file.

The lab then tasked me with making an executable that shows loops and conditionals, I created a new file using `touch` and edited it using `nano` again, the first image below shows what is inside the executable. From here I learnt the basics of Bash scripting, I learnt that adding a `$` at the beginning of an item would differentiate it from itself, if I did not put `$` infront of it, it would just `echo` out `i` instead of the variables 1 through 5 that I wanted. I also learnt about `do` and `done`, where done would close a boundary loop, whereas `fi` is a partner to `if` and closes conditional statements. I also learned more about regular expressions (regex) through the form of only accepting digits, `^[0-9]+$`, where `^` indicates the beginning of the string, `[0-9]` indicating the boundary of digits accepted, `+` means it can be inputted more than once and `$` to indicate the end of the string. the `!` at the beginning indicates that if the input is anything besides the regex, it would echo out the appropriate result. I also added a greater, equal or lesser than 10 using `-gt`, `-eq`, `elif`, `then` and `else`. The second image shows that it has ran successfully. The `for` loop in this instance would work by telling bash to repeatedly do the same code with the next variable. I then ran `chmod 777` to make sure it can be executed, and these are the results.

Finally, I had to do a system monitoring script, here I learnt that you don't need to use `touch` to be able to create the executable, I just had to use `nano` and it would create it automatically for me. So I went ahead and created resource_monitor.sh, this part of the lab was more straightforward, as I only had to run it once, from here I used `top`, however since `top` would usually output a ton of information and it would be in an interactive screen, I had to add `-b` to just print the output in plain text. To limit these updates, i added `-n 1` to make sure it only showed once. I then used the output of this and had a `|` to run it through `head -n 10` to make sure it only shows a limited amount of lines, the ones required at least so it isn't just outputting tons of information. I then added spacing in between the different resource information by adding `echo ""` between each of the different resource information. I then added `free -h`, which would basically show the RAM usage in a human readable way as `-h` is added, along with `df -h`, which shows the disk space. Then, I made sure the executable could be executed with `chmod 777` and the results are as shown.

From [Medium, 2023](https://medium.com/@guru_38941/shell-scripting-how-to-monitor-network-bandwidth-usage-with-a-bash-script-59b99db7239b), the `ifstat` command is used to monitor network bandwidth, the way to implement this would be by treating the current bandwidth as a variable, and if the variable is below or higher than an amount you deem is not good enough, you can `echo` a warning, `else` you could `echo` that everything is fine and the bandwidth is above the desired amount.

Automation is important for admins as it saves time by making repetetive tasks more consistent, all the while reducing human error. Using this lab as an example, instead of having to manually run `top` or `free -h`, an admin can just create a bash script with these to check on CPU and RAM, making system monitoring faster and more reliable.
